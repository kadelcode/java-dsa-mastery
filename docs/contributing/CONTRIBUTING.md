# Contributing Guide

Welcome to Java DSA Mastery.

This repository is a structured learning journey covering Java, Data Structures, Algorithms, Design Patterns, Problem Solving, and Software Engineering practices.

Even though this repository is currently maintained by a single contributor, all work follows professional engineering workflows.

---

# Development Workflow

## 1. Create an Issue

All work should begin with a GitHub Issue.

Examples:

* docs(java): add variables and data types notes
* feat(arrays): implement array traversal examples
* test(stacks): add stack unit tests

---

## 2. Create a Branch

Create a branch from `main`.

Branch naming convention:

<type>/<issue-number>-short-description

Examples:

feature/12-array-traversal
feature/15-linked-list-implementation
docs/21-java-generics-notes
test/33-array-unit-tests

---

## 3. Implement Changes

Keep changes focused on a single concern.

Examples:

* One algorithm implementation
* One documentation topic
* One test suite
* One refactoring task

---

## 4. Commit Changes

Follow Conventional Commits.

Examples:

feat(arrays): implement array traversal
docs(java): add variables guide
test(stack): add push and pop tests
refactor(queue): simplify enqueue logic

---

## 5. Open a Pull Request

Every change should be reviewed through a Pull Request, even when working alone.

Benefits:

* Maintains project history
* Encourages code review discipline
* Documents architectural decisions

---

## 6. Merge

Merge only after:

* Build passes
* Tests pass
* Documentation updated
* Acceptance criteria satisfied

---

# Code Standards

* Prefer readability over cleverness
* Write descriptive names
* Add tests when introducing behavior
* Document important concepts
* Keep methods small and focused

---

# Documentation Standards

Every significant topic should include:

* Concept overview
* Example code
* Complexity analysis (if applicable)
* References for further study

---

# Questions

If a future contributor is unsure how to proceed, open an issue for discussion before implementing changes.
